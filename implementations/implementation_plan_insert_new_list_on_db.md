# Save Manual Lists to `public.listas_do_usuario`

Insert a new row into `public.listas_do_usuario` immediately when a user manually creates a shopping list. On success the returned `id` (UUID generated by Supabase) replaces the temporary local ID so local state and the DB are in sync. Errors are surfaced as a SnackBar in the UI without blocking the local list creation.

## Proposed Changes

---

### Service Layer

#### [NEW] [supabase_list_service.dart](file:///home/vini/Documents/CheckListApp/lib/features/shopping_list/services/supabase_list_service.dart)

New class `SupabaseListService` with a single method:

```dart
Future<Map<String, dynamic>> insertManualList({
  required String userId,
  required String name,
  String? descricao,
  String moeda = 'BRL',
}) async { ... }
```

- Calls `Supabase.instance.client.from('listas_do_usuario').insert({...}).select().single()`
- Sets `origem: 'manual'`, `status: 'ativa'`
- Leaves `plano_alimentar_id`, `llm_geracao_id`, `finalizada_em` as `null`
- Returns the full inserted row (including the DB-generated `id`)
- Throws on Supabase errors (caller handles them)

### State / Controller

#### [MODIFY] [shopping_list_controller.dart](file:///home/vini/Documents/CheckListApp/lib/features/shopping_list/state/shopping_list_controller.dart)

- Add `SupabaseListService? _supabaseListService` and `String? _authenticatedUserId` fields
- Add `setSupabaseListService(SupabaseListService svc, String? userId)` setter — called from [main.dart](file:///home/vini/Documents/CheckListApp/lib/main.dart) via `ProxyProvider`
- Modify [addShoppingList(String name)](file:///home/vini/Documents/CheckListApp/lib/features/shopping_list/state/shopping_list_controller.dart#275-290):
  1. Create the [ShoppingList](file:///home/vini/Documents/CheckListApp/lib/features/shopping_list/models/shopping_list.dart#6-130) locally with `id: 'list-${timestamp}'` and add it to state as today (optimistic add)
  2. If authenticated and service is set, call `_supabaseListService!.insertManualList(userId, name)`
  3. On success: replace the local list's `id` with the Supabase-returned `id`; update `_activeListId` if needed; re-persist via repository
  4. On error: keep the local list (graceful degradation); let the exception propagate so the caller can show a SnackBar
- Return type changes from `Future<void>` to `Future<String>` (the final list ID)
